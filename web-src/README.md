# Исходники дизайна (Next.js) — не часть самого бота

Это исходный код Mini App-окна (сделан на v0, дизайн NEXUS), уже подключённый
к реальному бэкенду бота (`../webapp/server.py`). Сам бот эти файлы **не**
использует напрямую — он отдаёт собранную версию из `../webapp/static/`.

## Если просто хочешь запустить бота — этот файл можно игнорировать

`webapp/static/` уже содержит готовую собранную версию. Ничего собирать не
обязательно, чтобы бот работал.

## Если хочешь доработать дизайн

1. Установи зависимости (нужен Node.js 20+):
   ```bash
   cd web-src
   npm install
   ```
2. Редактируй компоненты в `components/miniapp/*.tsx` — вся логика похода
   в API уже вынесена в `lib/api.ts`, трогать её не обязательно, если просто
   меняешь вёрстку/стили.
3. Локально посмотреть можно через `npm run dev` (но Telegram WebApp SDK
   внутри обычного браузера работать не будет — `window.Telegram` не
   определён, часть функций будет падать в консоль. Для полноценного теста
   нужно открывать через сам бот).
4. Когда готов — собери статику:
   ```bash
   npm run build
   ```
   Next.js создаст папку `web-src/out/` — это и есть готовый сайт.
5. Скопируй содержимое `out/` в `../webapp/static/`, заменив старые файлы:
   ```bash
   rm -rf ../webapp/static/*
   cp -r out/* ../webapp/static/
   ```
6. Закоммить и запушь на GitHub как обычно — Render передеплоит сам.

## Если снова будешь дорабатывать дизайн через v0/другой ИИ

Отдавай ему на вход этот `web-src/` целиком (без `node_modules`), а не
файл `webapp/static/index.html` — то, что во `static/`, это собранный
Next.js-бандл, редактировать его напрямую бессмысленно, все правки
слетят при следующей сборке.

Если ИИ сгенерирует полностью новый проект (не патч к этому) — пришли мне
архив, я так же перенесу вызовы API из `lib/api.ts` в новую вёрстку.
